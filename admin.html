Panduan Resmi untuk Admin

Selamat datang,
Admin Panel! 👋

Dokumen ini adalah panduan lengkap cara menggunakan Pterodactyl Panel sebagai admin — mulai dari login, buat user, buat server, sampai kelola resource.

Role: Admin Pterodactyl v1.11 Bukan akses root/SSH Support via WA
Akses kamu adalah Admin Panel — kamu bisa buat & kelola user dan server, tapi tidak bisa akses SSH, ubah konfigurasi node, atau install egg baru. Untuk itu hubungi owner.

Apa yang Bisa Kamu Lakukan?

Kelola User

Buat, edit, reset password, dan hapus akun user yang kamu buat.

Kelola Server

Buat server baru, atur resource (RAM/CPU/disk), suspend, atau hapus server.

Atur Port

Assign port (allocation) ke server sesuai ketersediaan di node.

Database

Buat database MySQL untuk server-server yang membutuhkan.

Sub-user

Tambahkan sub-user ke server dengan permission terbatas.

Monitor Server

Lihat status, console, dan resource usage semua server.

Ingat: Semua tindakan sebagai admin tercatat di panel. Jangan buat user/server melebihi batas yang diberikan owner. Pelanggaran bisa berakibat akses dicabut.

Login & Navigasi

Cara masuk ke panel dan memahami layout dasarnya.

Cara Login

1

Buka URL Panel

Buka browser dan akses URL panel yang diberikan owner. Contoh: https://panel.domain.com

2

Masukkan Email & Password

Gunakan email dan password yang dikirim owner ke kamu. Pastikan tidak ada spasi di awal/akhir password.

3

Klik "Sign In"

Setelah berhasil, kamu akan diarahkan ke halaman utama panel.

4

Akses Admin Area

Klik avatar/nama kamu di pojok kanan atas → pilih "Admin Area" (ikon gear ⚙️). Di sinilah tempat semua fitur admin berada.

Kalau tidak muncul menu "Admin Area", berarti akun kamu belum di-set sebagai admin. Hubungi owner untuk konfirmasi.

Edit & Hapus User

Edit Data User

1

Admin → Users → klik nama user

2

Ubah data yang perlu

Kamu bisa ubah email, username, nama, dan password. Klik Update User setelah selesai.

Reset Password User

1

Buka halaman edit user

2

Isi field "Password" dengan password baru

Ketik password baru di kolom Password, lalu klik Update User. Beritahu user password barunya.

Hapus User

Perhatian! Hapus user akan menghapus semua server milik user tersebut secara permanen. Pastikan sudah backup data penting sebelum menghapus.
1

Admin → Users → klik nama user

2

Scroll ke bawah → klik "Delete User"

Konfirmasi popup yang muncul. Tindakan ini tidak bisa dibatalkan.

Suspend & Hapus Server

Suspend Server (Non-aktifkan Sementara)

Server yang di-suspend tidak bisa dijalankan, tapi data tetap ada.

1

Admin → Servers → klik server → tab "About"

2

Klik tombol "Suspend Server"

Server langsung di-stop dan user tidak bisa akses sampai di-unsuspend.

3

Untuk unsuspend: klik "Unsuspend Server"

Server bisa dijalankan kembali oleh user.

Reinstall Server

Reset file server ke kondisi awal (fresh install). Data server akan hilang.

1

Tab "About" → klik "Reinstall Server"

Konfirmasi popup. Proses sama seperti install ulang dari awal.

Hapus Server Permanen

Data hilang permanen! Semua file, database, dan konfigurasi server akan terhapus dan tidak bisa dikembalikan.
1

Admin → Servers → klik server → tab "About"

2

Scroll bawah → klik "Delete Server"

Ketik nama server untuk konfirmasi, lalu klik Yes, delete this server.

Allocations (Port)

Allocation adalah pasangan IP:Port yang bisa di-assign ke server.

Kamu hanya bisa memilih allocation yang sudah disiapkan owner. Menambah port baru ke node harus minta ke owner, karena perlu buka firewall di VPS.

Assign Port Tambahan ke Server

1

Admin → Servers → klik server → tab "Build Configuration"

2

Bagian "Allocations" → pilih port dari dropdown

Pilih allocation yang belum terpakai dari daftar. Port yang sudah dipakai server lain tidak akan bisa dipilih.

3

Klik "Update Build"

Port tambahan langsung tersedia di server. Server perlu di-restart agar port baru aktif.

Lihat Allocation yang Tersedia

1

Admin → Nodes → klik nama node

2

Tab "Allocation"

Semua port terdaftar akan muncul. Tanda Assigned artinya sudah dipakai. Tanda Unassigned artinya masih kosong.

Tips & Batas Wajar

Panduan agar akses admin kamu tetap aman dan tidak disalahgunakan.

Jangan Over-allocate

Beri resource sesuai kebutuhan. Jangan kasih RAM 10GB ke satu server kalau total node cuma 8GB.

Ingatkan User Backup

Data server bukan tanggung jawab owner jika hilang. Ingatkan user untuk rutin backup via panel.

Monitor Resource

Pantau server yang CPU/RAM-nya selalu tinggi. Bisa jadi ada script yang abuse resource.

Jangan Share Password

Jangan kasih password admin panel kamu ke siapapun. Kalau perlu kasih akses, buat akun terpisah.

FAQ

Server sudah dibuat tapi user bilang tidak bisa login?

Pastikan kamu sudah kirim email + password yang benar ke user. Cek ulang email yang dipakai saat buat akun. Coba reset password user dari Admin → Users.

Status server stuck "Installing" terus?

Biasanya karena image Docker sedang diunduh dan butuh waktu. Tunggu 5–10 menit. Kalau masih stuck, hubungi owner — mungkin ada masalah di Wings atau koneksi node.

Tidak ada allocation (port) kosong yang bisa dipilih?

Semua port sudah terpakai. Hubungi owner untuk tambah allocation baru ke node.

Bisa tidak ubah IP server?

Tidak bisa secara langsung — allocation terikat ke IP node yang sudah diset owner. Minta ke owner untuk pindah atau tambah IP baru.

Server error / crash terus setelah start?

Cek console server untuk pesan error. Biasanya karena: RAM kurang, file konfigurasi salah, atau versi Java tidak cocok dengan egg. Sesuaikan setting atau ganti egg.

Bisa tidak lihat log Wings / Docker dari panel?

Tidak. Log Wings/Docker hanya bisa dilihat owner via SSH ke node. Kamu hanya bisa lihat console output server dari panel.

Masalah yang tidak ada di FAQ ini? Langsung hubungi owner via WhatsApp. Sertakan nama server dan screenshot error supaya lebih cepat ditangani!
