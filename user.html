Panduan Resmi untuk User

Selamat datang,
Panel User! 👋

Dokumen ini adalah panduan lengkap cara menggunakan Pterodactyl Panel sebagai user — mulai dari login, kelola server, file manager, backup, sampai schedule otomatis.

Role: User Pterodactyl v1.11 Tanpa akses root/SSH Support via WA
Kamu login sebagai User — kamu hanya bisa mengelola server yang sudah diberikan ke akun kamu. Untuk request server baru, tambah resource, atau masalah teknis lainnya, hubungi admin.

Apa yang Bisa Kamu Lakukan?

Kontrol Server

Start, stop, dan restart server kamu kapan saja lewat panel.

Console

Lihat output log server secara live dan kirim command langsung dari browser.

File Manager

Upload, download, edit, rename, dan hapus file server lewat panel.

Backup

Buat backup server dan restore kapan saja jika ada yang error.

Schedule

Atur jadwal otomatis — misalnya restart server setiap malam jam 3.

Sub-user

Kasih akses terbatas ke teman untuk bantu kelola server kamu.

Penting: Kamu tidak bisa buat server sendiri, tambah port, atau ubah resource (RAM/CPU/disk). Semua itu harus minta ke admin. Kalau ada kendala, langsung hubungi via WhatsApp.

Start / Stop Server

Cara menghidupkan, mematikan, dan merestart server dari panel.

1

Klik server di Dashboard

Klik nama server yang ingin dikontrol. Kamu akan masuk ke halaman Console server tersebut.

2

Gunakan tombol kontrol di kanan atas Console

Start Stop Restart Kill

Gunakan Kill hanya kalau server hang total dan tidak bisa di-stop secara normal.

Selalu gunakan Stop (bukan Kill) untuk mematikan server secara normal — agar data tersimpan dengan benar. Kill akan memotong proses secara paksa dan bisa menyebabkan data corrupt.

Tabel Tombol Kontrol

TombolFungsiKapan Dipakai
StartMenghidupkan serverServer sedang Offline
RestartStop lalu Start otomatisSetelah update config, plugin
StopMatikan server dengan amanMau maintenance / update
KillPaksa matikan prosesServer hang / tidak responsif

Console & Command

Melihat output log server secara live dan mengirim command langsung dari browser.

Console di panel adalah console server, bukan terminal Linux. Kamu tidak bisa menjalankan command bash — hanya command yang dikenali aplikasi server (misalnya command Minecraft, command bot, dll).

Cara Kirim Command

1

Masuk ke Console server

Klik nama server → otomatis masuk ke tab Console.

2

Ketik command di kotak input bawah

Kotak teks di bagian paling bawah halaman Console. Tekan Enter untuk kirim.

Contoh Command Umum

Jenis ServerContoh CommandKeterangan
Minecraftop NamaPlayerJadikan player sebagai OP
Minecraftsay Halo semua!Kirim pesan ke semua player
MinecraftstopStop server dengan aman (sama dengan tombol Stop)
Minecraftwhitelist add NamaPlayerTambah ke whitelist
Node.js botrestart(tergantung implementasi bot-mu)
Console otomatis scroll ke bawah mengikuti log terbaru. Klik di area log jika ingin scroll manual ke atas untuk baca log lama.

SFTP

Akses file server menggunakan client SFTP seperti FileZilla atau WinSCP — lebih cepat untuk file besar.

Cara Dapat Info SFTP

1

Masuk ke server → tab "Settings"

Scroll ke bagian SFTP Details.

2

Catat info koneksi SFTP

FieldContoh Nilai
Hostpanel.domain.com
Port2022
Usernameu1.serverid (unik per server)
PasswordPassword akun panel kamu
3

Buka FileZilla / WinSCP

Masukkan Host, Port, Username, dan Password di atas. Gunakan protocol SFTP.

Rekomendasi client SFTP gratis: FileZilla (Windows/Mac/Linux) atau WinSCP (Windows). Untuk HP Android bisa pakai Cx File Explorer.

Backup

Buat backup server dan restore jika ada data yang hilang atau error.

Jumlah slot backup yang kamu punya tergantung dari yang diberikan admin. Kalau slot habis, hapus backup lama sebelum buat yang baru.

Buat Backup Baru

1

Masuk ke server → tab "Backups"

2

Klik "Create Backup"

Isi nama backup (opsional) dan daftar file/folder yang ingin di-exclude (opsional — biasanya biarkan kosong).

3

Tunggu proses selesai

Status backup akan berubah dari In Progress menjadi Completed. Jangan restart server saat backup berjalan.

Restore Backup

1

Klik ikon ⋮ (titik tiga) di samping backup

2

Pilih "Restore"

Panel akan meminta konfirmasi. Centang persetujuan bahwa file yang ada akan ditimpa, lalu klik Restore.

3

Tunggu proses restore selesai

Server akan otomatis stop, restore, lalu bisa kamu start kembali secara manual.

Perhatian: Restore akan menimpa semua file server saat ini dengan isi backup. Pastikan kamu sudah yakin sebelum restore. Buat backup baru terlebih dahulu jika ragu.

Aksi Backup Lainnya

AksiKeterangan
DownloadUnduh file backup (.tar.gz) ke komputer kamu
LockKunci backup agar tidak bisa dihapus secara tidak sengaja
DeleteHapus backup untuk bebaskan slot

Schedule (Jadwal Otomatis)

Atur task otomatis yang dijalankan secara terjadwal — seperti restart server tiap malam.

Buat Schedule Baru

1

Masuk ke server → tab "Schedules"

2

Klik "Create Schedule"

Beri nama jadwal dan atur waktu dalam format cron expression.

3

Tambahkan Task ke jadwal

Setelah schedule dibuat, klik nama schedule → klik New Task. Pilih jenis task dan isikan detail.

Contoh Cron Expression

CronArtinya
0 3 * * *Setiap hari jam 03:00 (tengah malam)
*/30 * * * *Setiap 30 menit sekali
0 */6 * * *Setiap 6 jam sekali
0 12 * * 1Setiap Senin jam 12:00
0 0 * * *Setiap hari tengah malam (jam 00:00)

Jenis Task yang Tersedia

TaskFungsiContoh
Send CommandKirim command ke console serversay Server akan restart dalam 1 menit!
Send Power ActionStart / Stop / Restart / Kill serverRestart otomatis
Create BackupBuat backup otomatisBackup harian
Contoh penggunaan keren: Buat schedule restart harian jam 3 pagi dengan 2 task berurutan — task 1: say Server restart dalam 5 menit!, tunggu 5 menit → task 2: restart. Player keburu siap-siap!

Network & Port

Informasi tentang port dan jaringan yang digunakan server kamu.

Sebagai user, kamu tidak bisa menambah atau mengganti port sendiri. Port sudah ditentukan admin saat server dibuat. Untuk request port tambahan, hubungi admin.

Cara Lihat Port Server

1

Masuk ke server → tab "Network"

Kamu akan melihat daftar port (allocation) yang terhubung ke server.

2

Bedakan Primary vs Additional Port

Primary adalah port utama server (yang digunakan player untuk connect). Additional adalah port tambahan (untuk plugin query, RCON, web, dll).

Two-Factor Authentication (2FA) menambah lapisan keamanan ekstra ke akun kamu.

1

Account → "Configure 2FA"

2

Scan QR code dengan Google Authenticator atau Authy

App authenticator akan generate kode 6 digit yang berubah tiap 30 detik.

3

Masukkan kode dari app untuk konfirmasi

2FA aktif. Setiap login akan meminta kode dari app authenticator.

Sangat disarankan aktifkan 2FA untuk keamanan server kamu. Simpan juga backup code yang diberikan saat setup — berguna kalau HP hilang.

API Key

API key digunakan jika kamu ingin mengontrol server via API (misalnya dari bot Discord atau script).

1

Account → "API Credentials"

2

Klik "Create New"

Beri nama key (deskripsi), isi allowed IPs (opsional, biarkan kosong untuk akses dari mana saja), lalu klik Create.

3

Salin API key sekarang!

Key hanya ditampilkan sekali. Jika hilang, hapus dan buat baru.

Contoh penggunaan API key
# Get server status via API
curl https://panel.domain.com/api/client \
  -H "Authorization: Bearer ptlc_YOUR_API_KEY_HERE" \
  -H "Accept: application/json"

Tips & Best Practice

Panduan agar server kamu tetap stabil, aman, dan efisien.

Rutin Backup

Buat backup minimal seminggu sekali, terutama sebelum update plugin atau konfigurasi besar.

Restart Rutin

Server yang jalan terus-menerus bisa bocor memori. Jadwalkan restart harian/mingguan via Schedule.

Pantau Resource

Cek CPU dan RAM secara berkala. Kalau selalu mendekati batas, hubungi admin untuk upgrade.

Bersihkan Log

Folder logs/ bisa membesar seiring waktu. Hapus log lama secara berkala agar disk tidak penuh.

Jaga Password

Jangan bagikan password akun panel ke siapapun. Gunakan fitur Sub-user jika ingin kasih akses ke orang lain.

Test Setelah Update

Selalu test server setelah update plugin atau konfigurasi. Jangan langsung tinggal — pantau console sebentar.

FAQ

Tidak bisa login ke panel?

Pastikan email dan password benar. Coba klik "Forgot Password" untuk reset via email. Jika email tidak ditemukan, hubungi admin karena akun mungkin belum dibuat atau emailnya berbeda.

Server status "Installing" terus?

Biasanya karena image Docker sedang diunduh pertama kali dan butuh waktu. Tunggu 5–10 menit. Kalau masih stuck, hubungi admin — mungkin ada masalah di node.

Server crash / error terus setelah start?

Cek console untuk pesan error. Penyebab umum: RAM kurang, file konfigurasi salah, plugin konflik, atau versi Java tidak cocok. Baca error di console dan sesuaikan konfigurasi.

Tidak bisa upload file karena ukurannya terlalu besar?

Panel ada batas ukuran upload. Untuk file besar (plugin, world, dll), gunakan SFTP dengan FileZilla atau WinSCP — tidak ada batas ukuran file.

Minta tambah RAM / CPU / Disk?

Hubungi admin via WhatsApp. Sertakan nama server dan alasan kenapa butuh upgrade resource. Admin akan melakukan upgrade dari sisi panel.

Player tidak bisa connect ke server Minecraft?

Pastikan server sudah Online (bukan Offline/Crashed). Cek port yang digunakan di tab Network dan pastikan player pakai format IP:PORT yang benar. Kalau port 25565, port bisa dikosongkan.

Lupa password akun panel?

Gunakan fitur "Forgot Password" di halaman login dan masukkan email akunmu. Jika email tidak valid atau tidak menerima email reset, hubungi admin untuk reset manual.

Backup gagal terus?

Kemungkinan slot backup sudah penuh. Hapus backup lama terlebih dahulu, lalu coba buat backup baru. Jika masih gagal, hubungi admin — mungkin ada masalah di storage node.

Masalah yang tidak ada di FAQ ini? Langsung hubungi admin via WhatsApp. Sertakan nama server dan screenshot error supaya lebih cepat ditangani!
